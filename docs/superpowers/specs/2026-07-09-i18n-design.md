# Мультиязычность UI (i18n)

**Дата:** 2026-07-09 · **Статус:** одобрено, в реализации

## Цель
Перевести UI-хром приложения на русский и английский через штатную Django i18n,
с возможностью добавлять другие языки одним `.po`-файлом. Дефолт — русский.

## Решение
Django gettext: `{% trans %}`/`{% blocktrans %}` в шаблонах, `gettext_lazy` в
Python, файлы `.po`/`.mo`, `LocaleMiddleware`, вью `set_language`, переключатель
RU/EN в сайдбаре (cookie).

**Исходный текст (msgid) — русский.** Текущие русские строки остаются как есть,
оборачиваются в trans; `locale/en/LC_MESSAGES/django.po` переводит RU→EN. Новый
язык = ещё один `<lang>.po` (перевод с русского).

### Что переводим (UI-хром)
- Все шаблоны: `base.html`, `jobs/feed.html`, `jobs/detail.html`,
  `jobs/_nav_item.html`, `analytics/analytics.html`, `tracks/list.html`,
  `tracks/edit.html`, `letters/_cover.html`, `screening/_screening.html`.
- Python UI-строки: `JobPosting.Status`/`BudgetType` labels, presenter-лейблы
  (риск, свежесть, статусы, «почасовая»), сообщения во вьюхах, лейблы/подсказки
  форм настроек.

### Что НЕ трогаем
- Сгенерированный контент: описания вакансий, письма, обоснования скоринга —
  это данные, они переводятся отдельной фичей (deep-translator).
- Telegram-бот (`apps/review`) — отдельная поверхность, позже.
- Внутренние/демо-строки (seed-команды, mock-эмбеддинги).

### Инфраструктура
- `settings.py`: `LocaleMiddleware` (после Session, до Common), `LANGUAGES=[ru,en]`,
  `LOCALE_PATHS=[BASE_DIR/"locale"]`, `USE_I18N=True` (уже есть), дефолт `ru`.
- `config/urls.py`: `path("i18n/", include("django.conf.urls.i18n"))`.
- `base.html`: `{% load i18n %}` + переключатель языка (форма на `set_language`).

### Пайплайн
`makemessages -l en` → заполнить `msgstr` (английский) → `compilemessages`.
gettext (`xgettext`/`msgfmt`) в системе есть — проверено.

### Тестирование
- Существующие 84 теста зелёные (обёртки не меняют логику).
- Тест: страница ленты рендерится по-английски при `?language=en`/cookie;
  по-русски по умолчанию.
- Каждая страница рендерится (200) в обоих языках.

## Порядок работ
1. Инфраструктура (settings, urls, base.html + переключатель) — вручную, проверить.
2. Обёртка строк по всем шаблонам и Python-модулям — параллельно (Workflow), по файлу на агента.
3. `makemessages` → перевод `en.po` → `compilemessages` → проверка обоих языков + тесты.

## Вне объёма (потом)
- Перевод Telegram-бота.
- Локализация форматов дат/чисел (Django `USE_L10N`/`USE_TZ` уже включены).
